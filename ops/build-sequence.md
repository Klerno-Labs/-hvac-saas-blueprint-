# Build Sequence

## Order of execution
1. Stage 0 spec freeze
2. Stage 1 validation assets
3. Stage 1 outreach and evidence collection
4. Stage 2 MVP architecture
5. Stage 2 MVP implementation
6. Stage 3 activation instrumentation and weekly habit feature
7. Stage 4 trust/conversion improvements
8. Stage 5 payments depth
9. Stage 6 collections automation
10. Stage 7 adjacency decision
11. Stage 8 platform expansion

## Rule
Nothing from a later stage gets built unless:
- the current stage gate issue is closed
- evidence is attached
- the repo milestone for the current stage is marked complete
